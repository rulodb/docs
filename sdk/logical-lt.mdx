## Syntax

<CodeGroup>

```ts TypeScript
value.lt(otherValue) â†’ result
```

</CodeGroup>

### Arguments

List of arguments to provide.

<ParamField body="otherValue" type="string | number" required>
  The value to compare with. The type of the value being compared depends on the value used for
  comparison.
</ParamField>

### Optargs

No optional arguments.

### Returns

<ParamField body="result" type="boolean">
  Returns `true` if the first value is less than the second value and `false` otherwise.
</ParamField>

## Behavior

- Performs less than comparison between two values.
- Uses RuloDB's comparison semantics for ordering and type handling.
- Can be chained with other logical operators for complex conditions.

## Notes & Caveats

- Comparison behavior depends on the data types being compared.
- String comparisons use lexicographic ordering.

## Example

### Filter users that are younger than 21

Filter for users where the age is under 21.

<CodeGroup>

```ts TypeScript
await r
  .table('users')
  .filter((doc) => doc.age.lt(21))
  .run(client);
```

</CodeGroup>

### Filter low value orders

Filter for orders where the value is less than 100.

<CodeGroup>

```ts TypeScript
await r
  .table('orders')
  .filter((doc) => doc.value.lt(100))
  .run(client);
```

</CodeGroup>
