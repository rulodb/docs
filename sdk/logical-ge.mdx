## Syntax

<CodeGroup>

```ts TypeScript
value.ge(otherValue) â†’ result
```

</CodeGroup>

### Arguments

List of arguments to provide.

<ParamField body="otherValue" type="string | number" required>
  The value to compare with. The type of the value being compared depends on the value used for
  comparison.
</ParamField>

### Optargs

No optional arguments.

### Returns

<ParamField body="result" type="boolean">
  Returns `true` if the first value is greater than or equal to the second value and `false`
  otherwise.
</ParamField>

## Behavior

- Performs greater than or equal comparison between two values.
- Uses RuloDB's comparison semantics for ordering and type handling.
- Can be chained with other logical operators for complex conditions.

## Notes & Caveats

- Comparison behavior depends on the data types being compared.
- String comparisons use lexicographic ordering.

## Example

### Filter users by age

Filter for users who are 21 or older.

<CodeGroup>

```ts TypeScript
await r
  .table('users')
  .filter((doc) => doc.age.ge(21))
  .run(client);
```

</CodeGroup>

### Filter by date

Filter for orders created on or after a specific date.

<CodeGroup>

```ts TypeScript
await r
  .table('orders')
  .filter((doc) => doc.createdAt.ge('2024-01-01'))
  .run(client);
```

</CodeGroup>
